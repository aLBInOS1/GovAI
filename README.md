## Описание проекта

GovAI — это не просто чат-бот для работы с гос. документами, а **инструмент для определения стратегического развития бюджетных направлений**. Проект демонстрирует, как с помощью LoRA-адаптации относительно небольшой модели (4B параметров) можно создать специализированного агента для:

- 📊 Анализа бюджетов по статьям КОСГУ с предложениями по оптимизации
- ⚖️ Оценки закупочной деятельности по 44-ФЗ и 223-ФЗ
- 📈 Разработки стратегий развития направлений (ЖКХ, образование, здравоохранение)
- 🔍 Анализа применимости ГЧП (115-ФЗ) для инфраструктурных проектов
- ⚡ Планирования энергоэффективных мероприятий (261-ФЗ)

### Новизна подхода

В отличие от универсальных LLM (GPT-4, Claude) и generic-решений (LM Studio + Qwen):

| Аспект | Универсальные агенты | Готовые решения | **GovAI** |
|--------|---------------------|-----------------|-----------|
| Верификация источников | ❌ Нет контроля | ❌ Нет | ✅ Контролируемый пайплайн |
| Специализированная лексика | ❌ Галлюцинации | ⚠️ Частично | ✅ Адаптация под ОГРН, КБК, ФЗ |
| Аудитопригодность | ❌ Нет | ❌ Нет | ✅ RAG + точные цитаты |
| Дообучение под гос. документооборот | ❌ Нет | ❌ Нет | ✅ LoRA-адаптация |
| Размер и требования | 🐳 70B+ параметров | 🐳 7B+ | 🚀 4B + LoRA-адаптер |

## Ключевые особенности

### 🎯 Специализация
- **Бюджетный анализ:** Распределение сокращений по КОСГУ с сохранением защищённых статей (210-212)
- **Закупочная аналитика:** Оценка НМЦК, выбор способа закупки, соответствие 44-ФЗ
- **Стратегическое планирование:** Цели, КПЭ, законодательная база, ожидаемые результаты

### 🔧 Архитектура
- **Базовая модель:** Qwen3.5-4B (Alibaba) — оптимальное соотношение качества/ресурсов
- **Адаптация:** LoRA (r=16, alpha=32) — снижение VRAM на 60% по сравнению с полным fine-tuning
- **Квантизация:** 4-bit NF4 — обучение на 2×T4 (15GB каждая)

### 📚 Датасет v11
- **1500+ примеров** 10 типов задач:
  - `budget_reduction` — сокращение бюджета с обоснованием по КОСГУ
  - `purchase_analysis` — анализ закупки по 44-ФЗ
  - `purchase_methods` — выбор способа закупки
  - `dynamics_analysis` — анализ динамики расходов
  - `region_comparison` — сравнение регионов
  - `procurement_planning` — планирование закупок
  - `strategy_development` — стратегическое планирование
  - `gpp_analysis` — анализ ГЧП (115-ФЗ)
  - `energy_audit` — энергоаудит (261-ФЗ)
  - `law_reference` — справка по законодательству

### Инференс с LoRA-адаптером
```python
from unsloth import FastLanguageModel

# Загрузка базовой модели + LoRA
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="unsloth/Qwen3.5-4B",
    adapter_name="path/to/govai-qwen-lora-v11",
    load_in_4bit=True,
)

# Генерация ответа
messages = [
    {"role": "system", "content": SYSTEM_PROMPT},
    {"role": "user", "content": "Проанализируйте закупку..."}
]
inputs = tokenizer.apply_chat_template(messages, return_tensors="pt")
outputs = model.generate(**inputs, max_new_tokens=2048)
```

## Результаты

### Метрики обучения (v11)
- **Платформа:** Kaggle, 2×T4 GPU
- **LoRA:** r=16, alpha=32, dropout=0.05
- **Эпохи:** 6 (early stopping с patience=3)
- **Время обучения:** ~3 часа
- **Final loss:** ~0.20 (в норме, без переобучения)

### Снижение галлюцинаций
- LoRA-адаптация на специализированном датасете **снижает галлюцинации** на гос. лексике
- Модель точнее цитирует статьи 44-ФЗ, БК РФ, 115-ФЗ
- Сохраняется thinking mode Qwen3.5 — цепочка рассуждений перед ответом

### Ограничения
- Модель требует проверки цитат законов (рекомендуется RAG-слой)
- Для production рекомендуется расширение датасета до 5000+ примеров
